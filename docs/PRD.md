# 產品需求文件 (PRD)：線上算命系統

## 1. 專案概述
- **背景與動機**：現代人生活壓力大，常好奇未來運勢或感到迷惘。傳統的實體算命或抽籤需要親自到廟宇，較為費時。開發「線上算命系統」，讓使用者隨時隨地能透過直覺的操作體驗線上抽籤、占卜，舒緩身心並獲得指引。
- **目標用戶**：對占卜、算命有興趣的社會大眾，或正遇到低潮需要尋求心理慰藉與方向提示的使用者。
- **核心價值主張**：提供簡單、快速、即時且具有娛樂與安定心靈作用的線上占卜服務，並能方便地保存及分享占卜結果。

## 2. 功能需求
系統將提供以下主要功能：
1. **線上算命、占卜與抽籤**：提供虛擬的抽籤或算命體驗，產出占卜結果。
2. **儲存算命結果**：允許使用者將覺得準確或有紀念價值的算命結果儲存至紀錄。
3. **展示算命結果**：在專屬頁面或歷史紀錄中查看過去的算命結果。
4. **重新占卜一次**：在獲得結果後，可快速一鍵重來，進行下一次抽籤或占卜。
5. **分享算命結果**：提供機制讓使用者能將有趣的結果分享給朋友。

### 使用者故事 (User Stories)
- **作為** 對占卜有興趣的使用者，**我希望** 可以在線上進行抽籤或塔羅牌占卜，**以便** 快速獲得今日運勢或人生問題的指引。
- **作為** 系統使用者，**我希望** 系統能夠自動或手動儲存我的算命結果，**以便** 日後需要時可以隨時回顧。
- **作為** 系統使用者，**我希望** 系統有一個明確的展示區來羅列我的算命歷史紀錄，**以便** 管理和查閱過去的測算。
- **作為** 正在進行占卜的使用者，**我希望** 在不滿意結果或想要測算其他問題時，能有一鍵「重新占卜一次」的功能，**以便** 無縫銜接後續操作。
- **作為** 系統使用者，**我希望** 能夠將特殊的占卜結果生成連結或複製文字進行分享，**以便** 與朋友討論和互動。

## 3. 非功能需求
- **技術限制**：
  - 伺服器端框架：使用 Flask。
  - 前端樣式：使用 Jinja2 模板引擎渲染 HTML。
  - 資料庫：使用 SQLite 來儲存使用者的算命紀錄。
- **效能與安全考量**：
  - 各頁面的載入速度應在 2 秒內完成。
  - 使用者資料及算命記錄的讀寫應具備基本防護（避免 SQL Injection）。
  - 若有會員系統，需確保個人占卜記錄的隱私，避免未驗證的使用者存取他人紀錄。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have（必須有）**：
  - 基本的線上算命/抽籤功能（亂數產生籤詩或結果）。
  - 結果展示與重抽一次的按鈕。
  - 將結果存進資料庫的基本功能。
- **Should Have（應該有）**：
  - 歷史存檔頁面，讓使用者能瀏覽過去所有的算命紀錄。
  - 簡單的登入/使用者識別機制，確保資料對應到正確的用戶。
- **Nice to Have（可以有）**：
  - 社群互動的分享按鈕（例如複製結果連結、Line/Facebook 分享）。
  - 精美的抽籤動畫、音效。
  - 多種占卜模式（如：行天宮籤詩、塔羅牌、每日星座運勢）。
