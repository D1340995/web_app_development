# 產品需求文件 (PRD)：食譜收藏夾系統

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，有時雖然喜歡烹飪，但卻面臨不知道要做什麼菜，或是食譜散落各處難以尋找的困擾。建立一個食譜收藏夾系統可以幫助使用者輕鬆尋找、集中管理他們喜愛的食譜，並獲得新的靈感。
- **目標用戶**：喜歡烹飪的人
- **核心價值主張**：方便搜尋食譜、收藏最愛的料理，並提供個人化的食譜推薦，讓烹飪變得更簡單有趣。

## 2. 功能需求
系統將提供以下主要功能：
1. **新增食譜**：允許使用者建立自己的食譜。
2. **列出材料**：在食譜頁面清楚列出所需材料。
3. **列出步驟**：以清晰的格式展示烹飪步驟。
4. **食譜收藏**：讓使用者可以收藏喜歡的食譜。
5. **食譜推薦**：主動向使用者推薦食譜。

### 使用者故事 (User Stories)
- **作為** 喜歡烹飪的人，**我希望** 能夠新增自己的食譜（包含名稱、簡介），**以便** 記錄並分享我的私房菜。
- **作為** 喜歡烹飪的人，**我希望** 在查看食譜時能清楚看到所需材料清單與份量，**以便** 我去超市採購。
- **作為** 喜歡烹飪的人，**我希望** 食譜能條列式顯示烹飪步驟，**以便** 我能在廚房一步一步跟著做，不會手忙腳亂。
- **作為** 喜歡烹飪的人，**我希望** 能夠將感興趣或經常做的食譜加入收藏夾，**以便** 日後快速尋找與查看。
- **作為** 喜歡烹飪的人，**我希望** 系統能根據我收藏的食譜或熱門排行推薦新食譜，**以便** 我發掘更多料理靈感，解決「不知道煮什麼」的煩惱。

## 3. 非功能需求
- **技術限制**：
  - 伺服器端框架：使用 Flask。
  - 前端樣式：使用 Jinja2 模板引擎渲染 HTML。
  - 資料庫：使用 SQLite 來儲存使用者資料與食譜內容。
- **效能與安全考量**：
  - 確保食譜搜尋與頁面加載的回應時間在合理範圍（例如 1-2 秒內）。
  - 防止 SQL Injection 與 XSS 攻擊，確保系統穩定與資料安全。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have（必須要有）**
  - 基礎食譜 CRUD（新增、瀏覽食譜內容包含材料與步驟）
  - 食譜搜尋（依食譜名稱關鍵字搜尋）
  - 食譜收藏功能
- **Should Have（應該要有）**
  - 簡單的食譜推薦區塊（例如：隨機推薦 3 道食譜）
- **Nice to Have（有的話很好）**
  - 使用者註冊與登入機制（讓每個人有獨立的收藏夾）
  - 食譜圖片上傳與展示
  - 食譜分類標籤（如：主食、甜點、湯品）
